## 🏛️ Skill 优先级仲裁规则（当多个 Skill 冲突时，按此顺序裁决）

> **设计原则（借鉴 KC_CLI "Thin harness, fat skills"）：**
> 骨架精简决定稳定性，内容丰富决定可用性。本规则是骨架层——只管"谁说了算"，不管"怎么做"。

**Tier 0 — 数据真相（不可覆盖）：**
- pdf-verifier / pdf-verifier-sec 从年报 PDF 提取的数字 > AKShare/yfinance 接口数据 > 报告中已有数字
- 当三者冲突时，以 PDF 原文为准；PDF 无法提取时回退到接口数据并标注 `[接口数据, 待人工确认]`

**Tier 1 — 诊断层 Skill（参数修正权）：**
- `ai-era-valuation` 输出的参数修正单 > `damodaran-valuation` 的默认参数假设
- 诊断层只修正参数，不改变建模框架本身

**Tier 2 — 建模层 Skill（方法论权）：**
- `damodaran-valuation` 的 DCF 框架和 WACC 推导规则 > 所有简化估值方法（EPV/相对估值/历史分位）
- `sotp-valuation` 在多元化公司场景下 > 单一 DCF（触发条件见 12.4）
- `peer-compare` 的同业对比结论只能辅助 DCF，不能覆盖 DCF 结论

**Tier 3 — 判断层 Skill（定性判断权）：**
- `financial-analysis` 的财务真实性判断（红旗信号）> 所有后续估值步骤
  - 若第四步判定"财务数据不可信"，估值结论自动降级为"仅供参考"
- `buffett-analysis` 护城河判断 + `duan-yongping-investment` 生意模式判断 → 影响第十三步评分权重，但不覆盖估值数字
- `position-management` 的仓位纪律 > 估值结论的操作暗示
  - 即使估值显示"深度低估"，若 position-management 规则禁止（如能力圈不足），仓位上限仍受约束

**Tier 4 — 辅助层 Skill（信息补充，无覆盖权）：**
- `industry-analysis`、`annual-report`（下载工具）、`pdf2skills` 产出的信息作为输入素材
- 辅助层不产生可以覆盖上层结论的判断

**冲突解决流程：**
```
1. 识别冲突：两个 skill 对同一参数/结论给出不同值
2. 比较 Tier：高 Tier skill 的结论自动胜出
3. 同 Tier 冲突：写出两者差异 + 选择哪个 + 理由（≥50字），记入报告正文
4. 记录：所有仲裁记录写入 session_log.json 的 arbitration_events 字段
```

## 各步骤执行规范

### 第一步：公司基本信息与术语通俗解释

**合格标准：** 一个投资小白读完能明白这家公司做什么

### 第四步：财务分析（六层框架）
**Full Mode 最低字数：2000字 | Quick Mode：800字**

**信号密度要求（与字数同等重要）：**
六层框架每层的论述必须包含：
- ≥ 1 条**量化证据**（具体数字：ROE/毛利率/资产负债率/FCFF等）
- ≥ 1 个**趋势锚点**（"过去 X 年从 X% 变为 X%"或"行业均值 X% vs 公司 X%"）
- ≥ 1 条**异常解释**（偏离行业均值或历史趋势的指标必须给出原因）

> 财务分析的价值不在罗列数字，而在数字背后的"为什么"。8000 字中有 3000 字是表格和数据，剩下的 5000 字应该是"这些数字说明什么"——后者才是信号密度。

**必须调用：** financial-analysis skill（Full Mode）

**Full Mode 六层缺一不可：**

| 层级 | 分析内容 | 最低字数 |
|------|---------|---------|
| 第一层：财务真实性 | 现金含金量、三表勾稽、十大红旗 | 300字 |
| 第二层：盈利质量 | 收入质量、毛利率、费用率趋势 | 300字 |
| 第三层：资产效率 | 周转率、营运资本 | 250字 |
| 第四层：财务安全 | 负债率、流动性、偿债能力 | 250字 |
| 第五层：成长能力 | 收入/利润/产量增速趋势 | 300字 |
| 第六层：股东回报 | ROE趋势、分红政策、回购 | 300字 |

**Quick Mode 仅做三层：** 财务真实性 + 盈利质量 + 财务安全（各约250字）

### 第五步：年报掘金及财务异常解释
**Full Mode 最低字数：1200字 | Quick Mode：300字**

**信号密度要求（与字数同等重要）：**
本步骤的价值在于"掘金"——从年报文字中提取管理层没明说但暗示的信息。每段掘金笔记必须包含：
- ≥ 1 条**量化锚点**（引用年报原文中的具体数字）
- ≥ 1 条**同比/环比变化**（"较去年增加/减少 X%"）
- ≥ 1 条"**反向解读**"（管理层没说但数字暗示了什么，或管理层强调了什么但数字不支持）

> 年报掘金的信号密度 = 每段笔记中"管理层说的" vs "数字实际说的"之间的信息差。没有信息差的笔记只是复述年报，没有分析价值。

**⚠️ Full Mode 特别强制要求：**

同比变化超过30%的指标，**每一个**都必须解释，格式：
```
【异常波动】XXXX年 XX指标 同比变化 XX%
原因：...（必须有具体原因，不能含糊）
影响：...
```

管理层讨论必须覆盖：战略表述、风险提示、产能计划、研发投入、重大事项

**Quick Mode 简化：** 仅覆盖最近两年年报的核心战略表述变化 + 主要风险因素变化，不做五年对比。

**📌 5.2 管理层战略表述五年对比分析（Full Mode 必须包含）**

对标中国平安v8版（已完成）的内容结构：

1. **五年经营方针对比表**（年报原文提炼的每年经营方针，含核心关键词、与上一年对比）
2. **战略表述五年演进**（四条主线逐一展开：综合金融、医疗养老、寿险改革、科技AI）
3. **风险因素五年演变**（消失了哪些风险、新增了哪些风险）
4. **【年报原文】对次年展望**（独立章节，严格区分原文与分析）
5. **利润增长点五年演进**（管理层对"增长从哪里来"的表述变化）
6. **管理层说了什么/没说什么**
7. **战略执行评分**（五年的判断力评估）

**核心方法论**：看"变化"不是看"表态"——被删掉的词、新出现的词，被量化的数字，才是管理层真正想传达的信息。

### 第六步：巴菲特护城河诊断（Full Mode 专用，Quick Mode 跳过）
**最低字数：800 字（信号密度优先于字数）**
**必须调用：** buffett-analysis skill
**五类护城河必须逐一判断（有/无/弱）并说明理由：**
- 资源禀赋护城河
- 规模效应护城河
- 成本优势护城河
- 区位优势护城河
- 品牌/多元化护城河

**信号密度要求（与字数同等重要）：**
每一类护城河的论述段必须包含：
- ≥ 1 条**量化证据**（市占率/毛利率差/客户留存率/续约率/翻新率等具体数字）
- ≥ 1 个**时间锚点**（"过去 X 年""自 20XX 年""未来 3 年监管即将"）
- ≥ 1 条**反方观察**（"但若 ____ 发生，护城河会被 ____ 侵蚀"）

⚠️ **避免凑字**：5 类护城河每类给出 3 个信号即可达标，写得太长不如写得精准。

**最终结论：** 护城河宽度评级（宽/窄/无）+ 一句话总结

### 第六点五步：管理层资本配置 5–10 年回测（Full Mode 专用，Quick Mode 跳过）

**最低字数：600 字**
**触发：** 所有公司都要做（巴菲特"一美元留存创造一美元以上市值"原则）
**必须包含的四张回测表：**

**1. 留存收益→市值创造测试（巴菲特"一美元留存创造一美元以上市值"原则）**
```
| 指标                                | 数值 |
| 过去 10 年累计留存收益（亿元）         | X    |
| 过去 10 年市值净增加（含分红，亿元）    | Y    |
| 留一块钱赚多少钱 = Y / X              | X.X  |
| 行业中位数对照（同期）                | X.X  |
| 评级（≥ 1.5 优秀；1.0–1.5 合格；< 1.0 价值毁灭）| ___ |
```

**2. 历次回购的均价 vs 当时 BV/合理价格**
```
| 回购年度 | 回购金额（亿）| 加权均价 | 当时 BV | P/B | 当时合理价格估算 | 创造/毁灭价值 |

**12.-1 公司类型分类（强制前置步骤，所有公司必须执行）**

在做任何估值之前，先判断公司属于哪个类型，因为这决定整条估值路径。对照下表逐一判断，**选中一个主类型、可兼选一个副类型**：

| 公司类型 | 判断标准 | 估值核心方法 | 主要挑战 |
|---------|---------|------------|---------|
| **A. 成熟稳定公司** | 盈利稳定≥5年，历史数据充足，增速<15% | 标准三阶段FCFF DCF | 增长假设不能过于乐观 |
| **B. 高速成长/初创公司** | 营收增速>30%，或尚未盈利，或上市<3年 | 情景DCF + 概率折扣 | 无历史锚点，需大量假设 |
| **C. 周期性公司** | 盈利随经济/大宗商品周期大幅波动（钢铁/煤炭/航运/普通化工等；精细化工/特种化工见下方化工细分规则判断归入 A 类还是 C 类）| 正常化盈利DCF，禁止用当期盈利 | 当期PE严重失真，需跨周期正常化 |
| **D. 金融服务公司** | 银行、保险、券商、信托 | 银行→FCFE/P/B；寿险→P/EV（**寿险板块 EV，非集团 EV**）；财险→P/B+COR；券商→FCFE DCF 为主、P/B-ROE 仅辅助 | 负债是"原材料"，不能用FCFF；ROE 不稳定时 P/B-ROE 失效 |
| **E. 大宗商品/资源公司** | 盈利主要由外部价格（铁矿石/原油/铜等）决定 | 价格情景DCF，含资源储量估算 | 外部价格不可控，需多情景 |
| **F. 新兴市场公司** | 注册/主营在高风险国家，或有显著国家风险敞口 | 调整后DCF，含国家风险溢价（CRP）| 汇率/政治风险难以量化（A 股**默认含中国 CRP**，不需作为副类型勾选）|
| **G. 困境/清算公司** | 持续亏损、债务违约风险高、或重组中 | 清算价值法 / 期权定价 | 现金流为负，DCF终值无意义 |
| **H. 平台/网络效应公司** | 单边或双边网络效应（用户/开发者/商户）+ 多业务并存 + 跨业务交叉补贴（典型：腾讯、美团、阿里、Pinduoduo）| **SOTP 为主**（各业务独立估值）+ 平台溢价（网络效应未变现部分作为补充情景）| 网络效应估值需用户网络效应 LTV/CAC + 数据资产估值；不可用单一 PE/PS 一刀切 |

**化工细分赛道分类规则（C 类 vs A 类判断）：**

> 化工行业横跨"强周期"到"弱周期"，不能一刀切。详细判断流程见 `valuation-art/references/cyclical-investing.md` 第六节。

判断核心不是"认证保护收入占比"这个单一数字，而是两个条件的交叉验证：

```
条件 1：产品是否需要客户认证（换供应商成本高）？
条件 2：公司是否经历过完整周期且 ROIC 在周期底部仍高于 WACC？
```

| 条件组合 | 分类 | 适用框架 |
|---------|------|---------|
| 条件1 ✓ + 条件2 ✓ | A 类（稳定溢价型）| ROIC 持续高于 WACC 决定 PE/DCF |
| 条件1 ✓ + 条件2 ✗ | C 类（特种但有周期）| 替代成本法 + ROIC 框架分部估值 |
| 条件1 ✗ | C 类（标准周期股）| 替代成本法 + 穿越周期 P/E |

典型公司：万华化学 MDI（条件1✓ + 条件2✓ → A 类，2015–2016 底部 ROIC 仍 12%+）；
华鲁恒升（条件1✗ → C 类）；新和成维生素（条件1✓ + 条件2✓ → A 类）。

**H 类与 D 类交叉处理规则：**

H 类是**父公司层面**的分类。在 12.4 SOTP 中，每个业务板块独立选用其对应的方法论：
- H + D 子板块（如腾讯金融科技）：该板块用 D 类金融估值
- H + C 子板块（如腾讯游戏成熟期）：该板块用稳定 PE
- H + B 子板块（如美团社区团购新业务）：该板块用 B 类情景估值

H 类的"平台溢价"是在 SOTP 加总后作为补充情景，不替代各板块的 SOTP 估值。

**副类型（可与主类型并列勾选，影响估值修正）：**

```

- 盈利能力价值（EPV = EBIT/WACC）
- 成长价值（简化DCF）
- 结论：当前价格贵/合理/便宜

**12.2 相对估值（必须完成）：**
- PE/PB/PS/EV-EBITDA（当前值 vs 历史均值 vs 行业均值）

**12.3 同业对比（peer-compare skill）：**

- ⚠️ 保险股P/EV必须用**寿险EV**，不能用集团EV（两者差一个数量级！）
  - 寿险EV ≠ 集团EV（集团EV含银行等非寿险业务）
  - 计算示例：集团P/EV = 股价 ÷ 集团每股EV（83.07元）
  - 计算示例：寿险P/EV = 股价 ÷ 寿险每股EV（寿险EV ÷ 总股本）
  - 两者意义不同：集团P/EV体现控股折价，寿险P/EV体现核心业务估值

**12.4 SOTP分部估值（sotp-valuation skill，如适用）：**
触发条件：公司为综合性金融集团、多元化产业集团、控股型公司、或新旧业务并存公司。

必须完成的步骤：
1. **业务拆分**：列出各业务板块的核心指标（EV/净资产/AUM等）
2. **板块估值**：各板块分别选择估值方法（P/EV→寿险，P/B→银行，P/AUM→资管，P/S→新业务）
3. **加总计算**：Σ各板块估值
4. **控股折价调整**：综合金融集团通常打8~9折（折价10%~20%）
5. **敏感性分析**：给出不同假设下的估值区间
6. **结论对比**：与当前市值对比，判断低估/高估方向

**SOTP输出格式（必须包含）：**
| 业务板块 | 核心指标 | 估值方法 | 参考倍数 | 估算价值 |
|---------|---------|---------|---------|---------|
| 寿险及健康险 | EV=X亿 | P/EV | 0.8x~1.2x | X亿 |
| 银行 | 净资产=X亿 | P/B | 0.4x~0.8x | X亿 |
| 产险及其他 | … | … | … | … |
| **合计** | | | | **X亿** |
| 控股折价 | 8~9折 | | | X亿 |
| **集团合理市值** | | | | **X亿** |

**⚠️ 重要区分：**
- 低估来源 = 控股集团折价 + 银行PB低 → 不是"寿险便宜"
- 寿险P/EV（1.0x~1.2x）往往并不比同业便宜，真正便宜的是控股结构

---

**保险股特殊规则：**
保险股不用DCF，用P/EV估值。因为保险公司有大量浮存金，DCF的现金流假设不适用。
- 寿险EV：保险公司清算价值，包含有效业务价值（VIFB）+ 经调整净资产（ANA）
- 集团EV：包含寿险EV + 银行净资产 + 产险净资产 + 其他业务 - 集团债务
- P/EV = 市值 ÷ EV，P/EV < 1说明市值低于内含价值（有安全边际）

|-----|--------------|
| DCF / 折现现金流 | "即把公司未来每年能赚到的钱，按一定的折现率折算成今天的价值，加总后即为公司的内在价值" |
| FCFF | "自由现金流（Free Cash Flow to Firm）：公司经营赚来的钱，扣除维持和扩张业务所必须投入的钱，剩下的才叫自由现金流——是真正属于全体资金提供者（股东+债权人）的钱" |
| WACC | "加权平均资本成本（Weighted Average Cost of Capital）：公司向股东和债主融资的综合成本，相当于'公司必须赚到这个回报率，才算没亏待出资人'" |
| 终值 / Terminal Value | "DCF预测期结束后，假设公司以稳定速度永续经营所产生的价值，通常占总估值的60–80%，是估值中最大的不确定性来源" |
| EV / 企业价值 | "Enterprise Value：相当于'买下整个公司要花多少钱'，等于市值加上净负债（债务减现金）" |
| Beta | "衡量这只股票相对大盘的波动幅度：Beta=1表示跟大盘同步；Beta=1.5表示大盘涨1%它涨1.5%，风险更高" |
| ERP / 股权风险溢价 | "投资者选择买股票而非无风险国债，要求额外获得的那部分回报补偿，全球均值约5–6%" |
| Sales-to-Capital Ratio | "每投入1元资本能带来多少元收入，衡量公司的资产轻重：比值越高=越轻资产，钱生钱效率越高" |
| ROIC | "投入资本回报率（Return on Invested Capital）：公司每投入100元运营资本，能赚回多少钱。ROIC持续高于WACC，意味着公司在真正创造价值" |
| P/EV | "市净内含价值比：只用于保险公司。P/EV = 市值 ÷ 内含价值，P/EV < 1意味着你用低于保险公司'清算价值'的价格在买它" |
| EPV | "盈利能力价值（Earnings Power Value）：假设公司不增长、只维持现状经营，应该值多少钱。是估值下限的参考锚" |

**报告写法示例（正确格式）：**
```
用WACC（加权平均资本成本，即公司综合融资成本，
本次取值9.2%，推导见下方12.1）对未来现金流进行折现……
```

**12.6 估值计算过程展示（Full Mode 强制要求；Quick Mode 用 EPV + 单情景简化 DCF 替代逐年表）**

估值报告不能只给结论，**必须展示推导过程**，让读者能够验证和质疑每一个数字。

**每个估值模块的计算必须包含以下三层：**

**第一层：关键参数说明**
列出所有输入假设及其来源/依据（不能凭空填数字）：
```
参数设定依据：
- 无风险利率 Rf = 2.3%（来源：2025年4月中国10年期国债YTM）
- 行业Unlevered Beta = 0.82（来源：damodaran-valuation assets/betas.xls，软件行业）
- 目标D/E = 15%（来源：公司近三年平均资本结构）
- ERP（A股默认，强制含 CRP）= 5.15%
    = 美国成熟市场 ERP 4.20% + 中国 CRP 0.95%
    （来源：Damodaran ERPApril26.xlsx + ctrypremApr26.xlsx）
→ 股权成本 Ke = 2.3% + 0.82×(1+0.85×15%)×5.15% ≈ 7.0%
→ 税后债务成本 Kd×(1-t) = (2.3%+1.2%)×(1-25%) = 2.6%
→ WACC = 7.0%×85% + 2.6%×15% ≈ 6.4%
```

> ⚠️ **A 股 ERP 必须包含中国 CRP**：所有 A 股公司默认 ERP = 美国 ERP + 中国 CRP；
> 仅对海外收入占比 > 50% 的公司才按地域加权。忽略 CRP 会让 Ke 系统性低估 70–100bps。
> 央企/大型国企的 ERP 调整规则见 damodaran-valuation/SKILL.md "国企特别处理"章节。

**第二层：年度预测展示（Full Mode 必须；Quick Mode 用简化版）**
DCF预测期的逐年数字必须以表格呈现：
```

|--------|---------|-----------|---------|---------|---------|---------|---------|---------|---------|
| 2025E  |         |           |         |         |         |         |         |         |         |
| 2026E  |         |           |         |         |         |         |         |         |         |
| …      |         |           |         |         |         |         |         |         |         |
| 2034E  |         |           |         |         |         |         |         |         |         |
```

**第三层：终值与股权价值推导**
```
终值计算：
  终值增长率 g = 2.3%
  ✅ g ≤ min(名义GDP长期增速, 无风险利率 Rf=2.3%)：满足
  ✅ g/Rf = 1.0 ≤ 1：满足
  理由：长期看名义 GDP 应收敛于 Rf；A 股当前 Rf ≈ 2.0–2.3%

  终值 FCFF = 2034 年 FCFF × (1+2.3%) = XX 亿
  终值 = XX 亿 ÷ (WACC - g) = XX 亿 ÷ (6.4%-2.3%) = XX 亿
  终值现值 = XX 亿 ÷ (1+6.4%)^10 = XX 亿
  终值占比 = 终值现值 ÷ EV = X% （强制 < 80%；> 80% 必须重新审视预测期长度或终值假设）

终值一致性检查（强制，三项缺一不可）：
  1. 预测期平均 g_revenue        = X.X%
  2. 预测期平均 ROIC            = X.X%
  3. 隐含 Reinvestment Rate (RR) = g / ROIC = X.X%
  4. 实际 RR = (CapEx - D&A + ΔNWC) / NOPAT 平均 = X.X%
  5. 偏差 = 隐含 RR - 实际 RR = X.X ppt
     - 偏差 ≤ 1.5 ppt：通过
     - 偏差 1.5–3 ppt：需在叙事中解释
     - 偏差 > 3 ppt 或 RR 偏差 > 15%：禁止使用，必须重做（典型问题：高 g + 低 RR + 不变 ROIC，三者必有虚的）



**ROIC 与 WACC 方向校验（强制）：**
  - 预测期平均 ROIC ≥ WACC → 价值创造，终值假设合理
  - 预测期平均 ROIC < WACC → 价值毁灭，终值年若 ROIC 仍 < WACC：
    - 强制解释："为什么市场应让这家公司继续经营而不是清算？"
    - 若无法解释 → 终值改用清算价值法，或降低预测期WACC/缩短预测期
  - ⚠️ 注意：这不是"让ROIC看起来高"，而是诚实地面对公司是否在创造股东价值

SBC（股权激励）处理（强制声明）：
  SBC（年化）= X 亿（占净利润 X%）
  处理方式（二选一，禁止"既不扣费用又不稀释"）：
    ☐ 路径 A：在 EBIT 中扣除 SBC 作为真实经济成本，FCFF 用基础股数
    ☐ 路径 B：EBIT 不扣 SBC，但用全摊薄股数（含 RSU/期权 TSM/可转债）
  本估值采用：[路径 A / 路径 B]，理由：____

企业价值汇总（含正确的股权桥接）：
  预测期 FCFF 现值之和  = XX 亿
  + 终值现值          = XX 亿
  = 企业价值 EV       = XX 亿
  - 净负债            = XX 亿（= 有息负债 - 货币资金 - 易变现金融资产）
  - 少数股东权益(NCI) = XX 亿（按公允价值/市值；金融控股集团必扣！）
  - 集团非核心责任   = XX 亿（如未确认的养老金、或有诉讼、表外担保）
  + 非经营性资产     = XX 亿（如可出售投资、超额现金、上市子公司股权按市值）
  = 归母股权价值      = XX 亿
  ÷ 稀释后总股本      = XX 亿股（含期权/RSU/可转债 TSM 转换）
  = 每股内在价值     = XX 元
  当前股价           = XX 元（获取时间：XXXX年XX月XX日）
  高估/低估幅度       = XX%
```

⚠️ **少数股东权益（NCI）扣除是金融控股集团、新能源车企、综合性集团的强制步骤**——
忽略 NCI 会让"归母每股内在价值"高估 5–15%（中国平安并表平安银行约 58% 持股，
NCI 占合并净资产 8–15%）。

⚠️ **稀释股数**必须包含：行使中期权、未行权 RSU、可转债 TSM 转换、潜在配股。仅用"总股本"
是错误的；高 SBC 公司可能因此低估稀释 5–10%。

⚠️ **表格中所有空格必须填入实际数字，不允许留白或写"见附件"。**

**12.7 历史价格验证（强制步骤，所有公司必须执行）**

估值结论不能悬浮在真空里。一个 PB=0.6x 或内在价值 5 元的结论，如果历史上从未出现过对应的成交价格，就必须解释"为什么这次会到那里"——否则结论只是一个内部自洽的数学游戏。

**⚠️ 本步骤是 12.6（估值计算过程展示）的必要补丁，不可跳过、不可合并。**

**12.7.1 基本要求（所有公司）**

完成估值计算后，必须将基准情景的估值结果与历史成交价做可行性核对：

| 核对项 | 要求 |
|-------|------|
| 基准估值 vs 历史 PB 区间 | 基准估值对应的 PB 是否在历史 PB 的区间内？ |
| 基准估值 vs 3/5年均价 | 基准估值与 3 年、5 年长期均价的偏离幅度是多少？ |
| 极端估值 vs 历史极值 | 最乐观/最悲观情景对应的估值，历史上是否出现过？ |

**判断规则：**
- 基准估值在历史区间内 → 注明"基准估值处于历史 XX%-XX% 分位，可接受"
- 基准估值低于历史最低 → **强制解释原因**（如"ROE 将首次跌破 XX%"或"行业结构发生根本变化"）
- 基准估值高于历史最高 → **强制解释原因**（如"估值方法论差异"或"隐性资产未被市场定价"）
- 数据不足（上市<3年） → 注明"历史数据不足以支撑坐标系判断"，不强行下结论

**12.7.2 金融公司专项：ROE-PB-隐含 Ke 历年表**

**适用范围：** 银行、保险、券商、信托等金融服务公司（即 12.-1 分类中的 D 类）。

> ⚠️ **公式适用边界（必读）：** 隐含 Ke 公式 `Ke = g + (ROE - g)/PB` 隐含了 Gordon Growth 的假设
> （ROE 永续稳定、派息率稳定、ROE > g）。在以下场景下公式失效，仅作为参考，不能作为决策依据：
> - **ROE 接近或低于 g 时**：分子 (ROE - g) 趋零或为负，Ke 失去经济意义
> - **ROE 多年波动 > 5pp**（典型如券商，历史 ROE 在 1–20% 间剧烈波动）：单年隐含 Ke 序列从负到 30% 不等，
>   均值/中位数都失去稳定性
> - **该公司处于亏损或恢复期**：BPS 被一次性损益扭曲
>
> 上述任一情形成立时，请改用 **P/B vs ROE 散点图**：横轴 ROE，纵轴 PB，把历史每个年度作为
> 一个点；用线性/对数回归得到"该公司在不同 ROE 水平下市场愿意给的 PB"，再判断当前 PB 是否
> 处于回归带的低位/高位/外面。这种方法对券商、有色金属、有色金融等高 ROE 波动公司更稳健。

金融公司必须建立完整的历史估值坐标系表，逐年回溯：

| ... | ... | ... | ... | ... | ... |

**隐含 Ke 计算公式：**
```

其中 g = 3%（固定），ROE 和 PB 取该年度实际值
```

**这张表必须揭示的三件事：**

**第一，PB 的历史区间和地板。** 最低 PB 出现在哪年、是多少。如果历史上 ROE 跌到 2% 时 PB 跌到 0.7x，就能判断"当前 PB=0.95x 只是接近底部区域而非底部本身"。

**12.8 市场预期诊断（强制步骤，Full Mode 所有公司必须执行；Quick Mode 仅做 12.8.1 + 12.8.4）**

绝对估值告诉你**"公允价值的下限"**——但市场长期支付以下溢价：(1) 流动性溢价、(2) 网络效应/平台溢价、
(3) 机构必配带来的指数权重溢价、(4) 未充分计入的可选性（optionality）。这些累计可达 15–40%，
是 Damodaran 估值常常显著低于市价的根本原因。

本节回答三个互补的问题：

1. **市场当前在为什么定价？**（反向 DCF）
2. **市场对这只股票历史上的"舒适区"在哪里？**（历史隐含 g 分位）
3. **绝对估值 × 历史分位 → 操作决策**（四象限矩阵）

**12.8.1 反向 DCF：市场在为什么定价？（强制，Full + Quick Mode 均须执行）**

固定 12.6 的 WACC、终值 g、终值 ROIC、利润率路径，**反向求解**使 DCF = 当前市值的关键变量
（通常是收入 CAGR 或终值利润率）。

```
| 指标                | 我的基础假设 | 市场隐含       | Delta      |
|--------------------|-----------|------------|------------|
| 未来 10 年收入 CAGR | X.X%      | X.X%       | ± X.X ppt  |
| 终值 EBIT 利润率    | X.X%      | X.X%       | ± X.X ppt  |
| 终值 ROIC           | X.X%      | X.X%       | ± X.X ppt  |
| 商业化期 Sales/Capital | X.X    | X.X        | ± X.X      |
```

**⚠️ 强制规则：报告不可直接引用市场隐含 g 的绝对值，只能引用 delta（你的假设 vs 市场假设的差值）。**
理由：反向 DCF 与 12.6 共享同一组基础参数（WACC、利润率路径等），绝对值的信息含量有限；
但 delta 在参数合理变动范围内通常稳定（WACC ±100bps 对 delta 影响通常 < 1ppt）。

**敏感性区间（Full Mode 强制；Quick Mode 可选）：**
使用 `scripts/reverse_dcf.py --sensitivity`，输出 WACC ±100bps × g终 ±50bps 的 3×3 敏感性表，
报告必须写出：
```
市场隐含收入 CAGR 稳健区间：X.X%–X.X%（WACC 敏感性 ±X.X ppt，g终 敏感性 ±X.X ppt）
→ Delta（你的 g vs 市场 g 稳健区间中值）= ± X.X ppt
```

**叙事翻译（强制写出 ≥ 200 字，Quick Mode ≥ 100 字）：**
> 市场目前在为 ____（具体故事）____ 定价。
> 我的判断是：市场过于乐观 / 悲观 / 合理，因为 ____。
> Delta X.X ppt 意味着：若我的判断正确，未来 12–24 个月的市场重估方向是 ____。

可使用本 skill 提供的脚本：`scripts/reverse_dcf.py`（见后）

**12.8.2 历史隐含增长率分位（Full Mode 专用）**

> ⚠️ **12.8.3（"估值溢价人格"）已合并到本节**。历史 implied_g 分位与历史溢价分位相关性 > 0.85，
> 无需对过去 10 年每年重做完整 DCF（工程成本极高且不可严格执行）。
> 用 implied_g 的 P25/P50/P75 作为溢价分位的代理即可。

把 12.8.1 的反向 DCF 在过去 10 年每个时点重做一次（用当时的财务数据 + 当时的股价），得到
`implied_g(t)` 时间序列：

```
| 时点    | 当时股价 | 当时一致预期 EPS/FCF | 反向求解 g_market(t) |
| -10 年  |        |                  |                  |
| -5 年   |        |                  |                  |
| -3 年   |        |                  |                  |
| -1 年   |        |                  |                  |
| 当前    |        |                  | ★ 当前值          |

**12.8.5 历史分位法的三个边界警告（必读，避免误用）**

> ⚠️ **警告 1：基本面拐点 → 历史分位失效**
> 诺基亚 2008–2010 历史分位一直显示"严重低估"，但其实是护城河永久性塌陷。
> 出现以下任一信号时，**禁止依赖历史分位**：
> - 核心业务关键指标连续 4 个季度趋势性恶化（不是周期，是趋势）
> - 行业出现颠覆性新进入者（传统燃油车 vs Tesla）
> - 监管/技术范式发生不可逆变化
>
> 此时必须重做基本面分析，并用"新均衡 DCF"重建估值。

> ⚠️ **警告 2：宏观流动性范式切换 → 必须校准 Rf 和分位**
> 茅台 2017–2021 高分位是合理的（LPR/Rf 持续走低、北上资金净流入、机构必配）；2022 后流动性
> 收缩，相同分位的预期收益完全不同。
> **校准规则：当前 Rf 与历史分位区间内 Rf 之差 > 100bps 时，分位需重新计算或重新解读。**

> ⚠️ **警告 3：样本量不足 → 不出独立分位表**
> - 上市 < 7 年：用同行业 4–5 家成熟公司的均值分位作交叉参考，本股票不出独立历史分位表
> - 上市 < 3 年：禁止使用历史分位法；同时按 position-management 规则禁止重仓

**填表规则（违反即返工）：**

1. **数量限制**：最少 3 个事件，最多 6 个（超过 6 个说明没筛选主次）
2. **"确定性"判断标准**：

每个事件的"事件"列开头必须标注分类标签：
- 📊 **[财报]** = 定期披露（季报/半年报/年报/业绩预告）
- 🏭 **[产能]** = 产线投产/扩产/技改/搬迁（蓬莱二期、HPPO二期等）
- 📜 **[政策]** = 行业监管/环保/税收/补贴政策变动
- ⚔️ **[竞品]** = 竞争对手新动作（新玩家进入、竞品降价、专利纠纷）
- 🌍 **[宏观]** = 汇率/利率/地缘/大宗商品价格等外生变量
- 💰 **[资本]** = 分红/回购/增发/可转债/股权激励等公司资本动作

**9. 优先级排序规则：**
表格按优先级从高到低排列。优先级 = 综合评估后的重要性排序，参考以下维度：
- 时间窗口：越近的事件越优先（1个月内 > 3个月内 > 6个月内）
- 确定性×影响幅度：高确定性×大幅影响 > 高确定性×小影响 > 低确定性×大幅影响
- 与kill point的关联度：直接触发kill point的事件优先级最高
- 是否已在DCF中price in：未price in的事件优先级 > 已price in的

**10. 催化剂衰减系数（量化评估工具）：**

对每个事件计算一个综合得分，用于交叉验证排序合理性：

```
衰减得分 = 确定性权重 × 影响幅度权重 × 时间折扣
  确定性权重：高=1.0, 中=0.6, 低=0.3
  影响幅度权重：取影响幅度的绝对值中位数（如 +8~12% 取10，±20% 取20）
  时间折扣：3个月内=1.0, 3-6个月=0.7, 6-12个月=0.4
  → 衰减得分 ≥ 8.0 = 高优先级催化剂
  → 衰减得分 3.0-8.0 = 中优先级
  → 衰减得分 < 3.0 = 低优先级（可考虑不纳入表格，用一句话提过即可）
```

> 衰减得分只是辅助排序工具，不是硬性门槛。但它能防止"把所有事件等权重罗列"的常见错误——6个月后的低确定性小影响事件不应与下个月的确定性大利好事件占同样篇幅。

**触发：** 所有公司都要做（即使绝对估值不达标也可能存在合理操作时机）

绝对估值告诉你**"公允价值的下限"**，但市场长期为流动性、网络效应、机构必配等因素支付溢价。
本步骤把"基本面 + 历史分位 + 估值"三层信号合并为可执行的操作纪律，避免"永远等到完美估值"。

**信号优先级（高 → 低，冲突时按此优先级执行）：**

> **第一优先级：E 类禁止信号（任一触发即清仓/减至观察仓，无视其他类别信号）**
> - 13.0 反向论文中任一 kill point 触发 → 强制 E
> - 财务造假或重大违规暴露 → 强制 E
> - 反向 DCF delta 显示市场比你乐观（delta > 阈值）→ 强制 E
>   - 阈值按公司类型区分：成熟公司 3 ppt，成长公司 5 ppt，周期公司 3 ppt
>
> **第二优先级：A 类（仅在 E 类全部不触发时生效）**
> **第三优先级：B / C / D 类按其各自条件**
>
> **冲突输出格式（强制）：**
> 当 A 类信号触发但被 E 类压制时，必须输出：
> "信号 A 触发但被 E 类压制，理由：____。建议：先观察 1 个季度，等市场预期与你的判断重合再行动。"

**触发器规则（A–E 五类）：**

```
A. 估值面 + 历史分位双低（12.8.4 四象限的 ★★★ / ★★ 区）
   触发：绝对估值低估 + 历史隐含 g 处于 P25 以下 + 反向 DCF 显示市场比你悲观 ≥ 3 ppt
   操作：可加仓至确定性分级允许的上限，无需额外催化剂

B. 估值面合理 + 强基本面催化（满足任一）
   - 财报超预期且 12 个月内还有 ≥ 2 次大概率超预期
   - 重大产品/订单落地（已在 6 个月内确定）
   - 监管/政策清障（垄断、合规、上市等）
   - 大股东/管理层显著增持（≥ 流通市值 0.5%）
   操作：试仓，仓位上限 ≤ 满仓的 50%
   止损：-15% 强制评估论文是否仍成立

C. 估值面合理偏高 + 双重确认（绝对估值高估 0–15%）
   - 基本面拐点向上（≥ 2 个季度数据支持）
   - 技术面破年线/突破密集筹码区
   - 机构持仓比例从历史低位回升
   操作：跟随仓位（不超过中等确定性的下限），止损纪律严格（-15% 出局）

D. 估值面高估，仍可"对冲式持有"（绝对估值高估 15–30%）
   - 仅当原有仓位的成本远低于当前内在价值时
   - 操作：不卖，但停止加仓；股价若超过历史溢价 P85 → 减仓 1/3

E. 禁止信号（必须减仓至观察仓或清仓）
   - 反向 DCF delta 显示市场比你乐观超过阈值（成熟 3 / 成长 5 / 周期 3 ppt）
   - 当前估值 + 历史分位均在 P85+
   - 13.0 反向论文中任一 kill point 触发
   - 财务造假或重大违规暴露
```

**强制输出格式：**

```
