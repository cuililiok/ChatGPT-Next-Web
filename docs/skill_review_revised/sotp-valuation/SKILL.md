# 分部加总估值（SOTP）框架

## 核心理念

> "如果一家公司由 5 个业务组成，每个业务的现金流特征、风险特征、增长特征都不同，
> 那么把它们打包用一条 DCF 折现就是把信号混入噪声。"

SOTP 的目的不是把估值变得更复杂，而是承认**多业务公司的真实价值需要分别评估再加总**——
合并报表丢失了大量结构信息，而结构信息恰恰是估值差异的来源。

---

## 何时必须用 SOTP，何时不用

**必须用：**
- 各业务板块的"估值锚"不同（保险 → P/EV，银行 → P/B，资管 → P/AUM，互联网 → DCF/PS）
- 板块之间盈利波动相关性低
- 任一板块占合并收入或净资产 > 15%
- 存在显性的隐性资产（如地产持仓的市价 vs 账面价、上市子公司股权的市值 vs 权益法账面）

**不要用：**
- 公司业务高度同质（一个白酒主业 + 一点点子公司）
- 合并子公司无独立财务披露（缺数据，强行 SOTP 是凭空捏造）
- 业务间存在不可分离的协同价值（如品牌共享、渠道共享，拆开估值会双重低估）

## 第一步：业务拆分

### 1.1 拆分原则

按"是否有独立的现金流特征"拆，**不是按事业部组织架构拆**：

- 一个业务的收入、毛利率、资本开支节奏与其他业务显著不同 → 独立分部
- 仅仅是地理或客户细分而盈利模式相同 → 合并为一个分部

### 1.2 拆分输出

```
| 板块         | 收入(亿) | 净利润(亿) | 净资产(亿) | 占比% | 估值锚指标         |
|------------|--------|---------|---------|-----|----------------|
| 寿险及健康险      | XXX    | XXX     | (EV=XXX) | XX% | 内含价值 EV / 新业务价值 |
| 财产保险        | XXX    | XXX     | XXX     | XX% | 净资产 / 综合成本率 COR  |
| 银行          | XXX    | XXX     | XXX     | XX% | 净资产 / ROE       |
| 资产管理        | XXX    | XXX     | (AUM=XXX) | XX% | AUM / 管理费率      |
| 互联网/科技业务    | XXX    | (亏损 XXX) | -       | XX% | 收入 PS / 用户数      |
| 上市子公司持股     | -      | (权益法 XXX) | -      | XX% | 持股比例 × 子公司当前市值  |
| 集团总部 / 内部交易 | (XXX)  | (XXX)   | (XXX)   | -   | 直接抵销           |
```

### 1.3 数据来源（A 股）

- **板块财务数据**：年报"分部信息"附注（财政部要求 ≥ 10% 收入/利润板块单独披露）
- **保险 EV/VNB**：年报"内含价值报告"
- **银行子公司**：上市子公司直接取市场价 × 持股比例；非上市用 P/B
- **资管 AUM**：年报"管理资产规模"披露
- **持股市值**：实时 AKShare `stock_zh_a_spot_em()` × 持股比例

## 第二步：各分部估值方法选择

| 分部类型 | 首选方法 | 公式 | 关键参数 | 注意事项 |
|------|------|------|------|------|
| **寿险公司** | P/EV | 市值 = EV × 倍数 | 倍数：A 股寿险历史 0.6–1.2x，看 ROEV 和 NBV 增速 | 用**寿险板块 EV**，不是集团 EV（集团 EV 含银行等非寿险业务） |
| **财产保险** | P/B + COR | 市值 = 净资产 × 倍数 | COR < 100% 的财险给 1.0–1.8x；COR > 100% 给 0.6–1.0x | 用 5–10 年平均 COR，避免单年灾害扭曲 |
| **商业银行** | P/B 配 ROE | 市值 = 净资产 × P/B(ROE) | 倍数 = (ROE - g) / (Ke - g)，A 股银行常见 0.4–0.9x | ROE 需用预期 ROE（监管收紧 → 下调） |
| **证券/资管** | FCFE DCF（首选）+ P/AUM 交叉 | EV = Σ FCFE/(1+Ke)^t | 半衰 5 年指数加权 ROE 正规化 | P/B-ROE 在 ROE 接近 Ke 时失效 |
| **公募/被动管理** | P/AUM 或管理费 × DCF | 市值 = AUM × (3‰–8‰) | 管理费率 + 规模 + 增长 | 主动 vs 被动管理 AUM 估值倍数差 2–3 倍 |
| **互联网/科技平台** | DCF 或 EV/Sales | 收入 × 行业 PS（亏损期）或 DCF | 留存率、ARPU、获客成本 | 亏损期勿用 P/E |
| **传统制造/消费** | DCF + EV/EBITDA | 标准 DCF | 行业 unlevered β + WACC | 周期股需正常化盈利 |
| **资源/大宗** | 价格情景 DCF + NAV | 储量 × 净价 | 价格曲线、储量、提取成本 | 多价格情景加权 |
| **房地产** | NAV（净资产价值） | 资产市价 - 负债 | 市价用市场租金或可比成交 | 不用 P/E |
| **上市子公司股权** | 直接市值法 | 持股比例 × 子公司流通市值 | 流通市值 vs 全市值差异 | 是否享有控制权决定是否打折 |
| **未上市子公司** | 行业可比倍数 × 业绩 | 利润 × 行业 P/E 中位数 | 同行业上市公司中位数 | 流动性折扣 20–30% |
| **新业务/亏损部门** | 收入 PS 或 EV/Sales | 收入 × 行业 PS | 行业可比 PS 中位数 | 若长期不盈利可考虑赋负值 |

## 第三步：加总计算

### 3.1 加总公式

```
SOTP 企业价值 = Σ (各板块估值)
              - 集团层债务（仅集团母公司层面，非合并债务！）
              - 集团总部费用资本化值（若总部为净成本中心）
              - 少数股东权益（按板块对应比例）
              + 集团层现金（剔除业务必要现金后剩余）
```

### 3.2 易错点

- **集团层 vs 板块层债务**：合并债务包括各板块经营性负债（如保险责任准备金、银行存款），
  这些已经在各板块估值中扣除了，不能在 SOTP 加总时再扣一次。**只扣集团母公司层面的有息负债。**
- **板块层净资产**：分部估值用的净资产应剔除内部交易、内部持股；用"分部权益"而非"持股账面值"。
- **少数股东权益**：分板块估值时每个板块的"全口径"估值已包含少数股东，加总后必须扣除少数股东应得部分。

## 第四步：控股折价（Holding-Company Discount）

### 4.1 为什么打折

控股集团相对于其分部市值的简单加总，市场会打折，原因：
1. **税收摩擦**：分部利润上交集团再分配，中间有股息税或重组税
2. **代理成本**：集团管理层未必把资本配置给最优分部，可能交叉补贴
3. **披露折扣**：合并报表掩盖分部表现，投资者要求信息不对称溢价
4. **流动性折扣**：综合公司不在任何指数权重最大，机构关注度低
5. **资本配置摩擦**：板块间资本调配受到监管或商业逻辑限制

### 4.2 折扣率参考

| 集团类型 | 典型折扣率 | 触发上调（折扣更深）的因素 |
|------|------|------|
| 综合金融集团（保险+银行+资管） | 10–20% | 控股股东占比过高、交叉补贴明显、多次重大并购失败 |
| 多元化产业集团 | 15–25% | 业务相关性低、缺乏战略协同 |
| 平台型公司（多业务网络效应） | 5–15% | 网络效应清晰时折扣较浅；多业务相互独立时折扣较深 |
| 控股投资公司 | 20–35% | 类似封闭式基金，长期看 NAV 折扣 |
| 资源型综合集团 | 15–25% | 商品价格周期、储量披露不透明 |
| 优秀资本配置者（如伯克希尔） | 0–10% | 历史长期 ROIC 远高于行业、CEO 资本配置记录优秀 |

**A 股特例：**
- 中国平安（保险+银行+科技）：历史折扣 15–25%
- 招商局集团（多业务）：历史折扣 20–30%
- 复星国际（投资型控股）：历史折扣 25–40%

### 4.3 折扣的反向操作

折扣不是永远的——若集团宣布重大重组（分拆上市、分部 IPO、出售非核心资产），折扣会快速收敛。
本框架建议把"重组催化剂"作为单独章节列出。

## 第五步：敏感性分析

### 5.1 关键变量

每个分部估值结果对至少 1 个变量敏感，集中起来做敏感性表：

```
| 变量            | 基准 | 悲观 (-15%) | 乐观 (+15%) | 对集团估值影响 |
|---------------|----|----------|----------|---------|
| 寿险 P/EV 倍数     | 1.0x | 0.85x    | 1.15x    | ±X 亿     |
| 银行 ROE 假设      | 11% | 9%       | 13%      | ±X 亿     |
| 互联网业务 PS 倍数    | 3x | 2x       | 4x       | ±X 亿     |
| 控股折价         | 15% | 25%      | 5%       | ±X 亿     |
| **集团估值区间** | XXX 亿 | XXX 亿 | XXX 亿 | (悲观-乐观区间宽度 ÷ 基准 = X%) |
```

### 5.2 区间合理性检查

- 悲观-乐观区间宽度 / 基准 < 30%：估值过于自信，可能漏掉了关键变量
- 悲观-乐观区间宽度 / 基准 > 80%：估值过于发散，应增加约束或重新审视假设

## 第六步：与当前市值对比，识别低估来源

### 6.1 对比格式（强制输出）

```
                    SOTP 估值     当前市值      差值       差值来源解读
基准情景 SOTP        X 亿         Y 亿         Y - X      ─────
─────────────────────────────────────────────────────
低估来源拆解：
  - 寿险板块定价 vs 历史中位数        ±X 亿 (XX%)
  - 银行板块 P/B vs 历史中位数        ±X 亿 (XX%)
  - 互联网/新业务市场未充分定价     ±X 亿 (XX%)
  - 上市子公司持股折让              ±X 亿 (XX%)
  - 控股折价历史水平偏高/偏低         ±X 亿 (XX%)
  - 净现金/未充分利用资本            ±X 亿 (XX%)
─────────────────────────────────────────────────────
低估比例 = (SOTP - 市值) / 市值 = X%
```

### 6.2 关键判断

- **低估若主要来自"控股折价过深"** → 需要重组催化剂触发，时间不确定
- **低估若主要来自"某板块估值偏低"** → 等行业回暖即可
- **低估若主要来自"隐藏资产未定价"** → 需要管理层主动披露/分拆动作

## 与 14-step-analysis 的衔接

14-step-analysis 第 12.4 节触发本 skill 后，本 skill 的输出（SOTP 估值数字、低估来源、敏感性）
应作为整个 12 章估值结论的核心，并与：
- **12.6 标准 DCF 估值** 交叉对比（若公司单一主业 > 70%，DCF 与 SOTP 应大致一致）
- **12.7 历史价格验证** 联合判断（SOTP 估值是否在历史 PB / PE 区间内）
- **12.8 市场预期诊断**（见 14-step-analysis SKILL）联动判断

## 常见错误

| 错误 | 后果 | 纠正方式 |
|------|------|---------|
| 用集团 EV 算寿险 P/EV | 寿险估值虚低 1 个数量级 | 用寿险板块 EV |
| 把合并债务全部扣 1 次 | 双重扣除 | 只扣集团母公司层面有息负债 |
| 不扣少数股东权益 | 估值高估 5–15% | 在 SOTP 加总后扣除 |
| 互联网亏损业务赋负值 | 长期看会回到 0 但短期失真 | 用 EV/Sales 设置下限 |
| 不打控股折价 | 估值长期高于市价无法收敛 | 强制 5–25% 折扣 |
| 折价用历史平均不结合公司具体情况 | 错过结构性变化 | 重大重组发生时折扣应快速调整 |
| 板块间不抵销内部交易 | 收入和利润双重计算 | 严格按"对外收入"口径 |
