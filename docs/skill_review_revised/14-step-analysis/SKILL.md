### 第六点五步：管理层资本配置 5–10 年回测（强制，介于第六步和第七步之间）

**最低字数：600 字**
**触发：** 所有公司都要做（伯克希尔/巴菲特式硬测试）
**必须包含的四张回测表：**

**1. 留存收益→市值创造测试（巴菲特"一美元留存创造一美元以上市值"原则）**
```
| 指标                                | 数值 |
| 过去 10 年累计留存收益（亿元）         | X    |
| 过去 10 年市值净增加（含分红，亿元）    | Y    |
| 留一块钱赚多少钱 = Y / X              | X.X  |
| 行业中位数对照（同期）                | X.X  |
| 评级（≥ 1.5 优秀；1.0–1.5 合格；< 1.0 价值毁灭）| ___ |
```

**2. 历次回购的均价 vs 当时 BV/合理价格**
```
| 回购年度 | 回购金额（亿）| 加权均价 | 当时 BV | P/B | 当时合理价格估算 | 创造/毁灭价值 |
| ...    | ...         | ...    | ...   | ... | ...            | ...          |
| 合计    |             |        |       |     |                | 累计 ±X 亿     |
```

**3. 历次重大并购的减值情况**
```
| 并购年度 | 标的         | 对价（亿）| 当年商誉（亿）| 截至 2025 年累计减值（亿）| 减值/对价比 | 评级 |
| ...    | ...         | ...     | ...        | ...                  | X.X%       | ___ |
```
减值/对价比 < 10%：合格；10–30%：警示；> 30%：管理层并购能力存疑

**4. IPO 以来净增发 - 净回购股本（稀释/反稀释指数）**
```
IPO 时总股本：X 亿股
当前总股本：Y 亿股
扣除拆分/送股影响后的"实质股本变化"= Y - X = ±X 亿股
   - 净回购（负值）：股东友好型管理层
   - 净增发（正值）：稀释型管理层（除非每次增发都用于高 ROIC 项目）
```

**结论部分必须明确回答：**
- 这位管理层在过去 10 年是创造股东价值还是毁灭股东价值？
- 在低估值区间是否敢于回购？在高估值区间是否克制增发？
- 历次并购的减值记录是否支持其继续从事并购？

⚠️ **本步骤是第七步"段永平生意模式诊断"中"管理层是否优秀"维度的**硬测试**，
不能用定性表述（"诚信记录良好""经营能力强"）替代——必须有数字。

**12.-1 公司类型分类（强制前置步骤，所有公司必须执行）**

在做任何估值之前，先判断公司属于哪个类型，因为这决定整条估值路径。对照下表逐一判断，**选中一个主类型、可兼选一个副类型**：

| 公司类型 | 判断标准 | 估值核心方法 | 主要挑战 |
|---------|---------|------------|---------|
| **A. 成熟稳定公司** | 盈利稳定≥5年，历史数据充足，增速<15% | 标准三阶段FCFF DCF | 增长假设不能过于乐观 |
| **B. 高速成长/初创公司** | 营收增速>30%，或尚未盈利，或上市<3年 | 情景DCF + 概率折扣 | 无历史锚点，需大量假设 |
| **C. 周期性公司** | 盈利随经济/大宗商品周期大幅波动（钢铁/煤炭/航运/普通化工等，但**精细化工/特种化工/化工新材料因认证壁垒应归入 A 类**）| 正常化盈利DCF，禁止用当期盈利 | 当期PE严重失真，需跨周期正常化 |
| **D. 金融服务公司** | 银行、保险、券商、信托 | 银行→FCFE/P/B；寿险→P/EV（**寿险板块 EV，非集团 EV**）；财险→P/B+COR；券商→FCFE DCF 为主、P/B-ROE 仅辅助 | 负债是"原材料"，不能用FCFF；ROE 不稳定时 P/B-ROE 失效 |
| **E. 大宗商品/资源公司** | 盈利主要由外部价格（铁矿石/原油/铜等）决定 | 价格情景DCF，含资源储量估算 | 外部价格不可控，需多情景 |
| **F. 新兴市场公司** | 注册/主营在高风险国家，或有显著国家风险敞口 | 调整后DCF，含国家风险溢价（CRP）| 汇率/政治风险难以量化（A 股**默认含中国 CRP**，不需作为副类型勾选）|
| **G. 困境/清算公司** | 持续亏损、债务违约风险高、或重组中 | 清算价值法 / 期权定价 | 现金流为负，DCF终值无意义 |
| **H. 平台/网络效应公司** | 单边或双边网络效应（用户/开发者/商户）+ 多业务并存 + 跨业务交叉补贴（典型：腾讯、美团、阿里、Pinduoduo）| **SOTP 为主**（各业务独立估值）+ 平台溢价（网络效应未变现部分作为补充情景）| 网络效应估值需用户网络效应 LTV/CAC + 数据资产估值；不可用单一 PE/PS 一刀切 |

**副类型（可与主类型并列勾选，影响估值修正）：**

```
☐ A/H 双重上市（标注哪边交易、AH 价差当前位置）
   - 修正：基础估值用 A 股，再调整流动性差和折扣 5–25%
☐ 中概股 ADR + VIE 结构（标注 VIE 风险敞口）
   - 修正：在 EV 上叠加 5–15% 风险折扣（基于 VIE 结构占合并主体的比例 + 历史监管事件估算）
☐ H 股小盘流通（自由流通市值 < 50 亿港币）
   - 修正：流动性折扣 5–15%（参考同行业大盘 H 股估值倍数差）
☐ 大型央企/国企 + 国家战略行业（油、电、电信、银行、军工）
   - 修正：见 damodaran-valuation/SKILL.md "国企 Beta 与 ERP 调整"章节
   - levered beta × 0.85（隐含国家担保）
   - ERP 加 0.5–1.0% 政策风险溢价（决策不完全市场化）
☐ AI 相关公司（即使主类型不是 H）
   - 强制触发 12.0 AI 估值诊断
```

**分类结论格式（强制输出）：**
```
公司类型判断：主类型 = [A/B/C/D/E/F/G]，副类型 = [如有]
判断依据：[2-3句具体说明为什么选这个类型]
估值路径：[根据类型确定下方使用哪些估值方法，哪些跳过]
特殊处理说明：[如周期股需正常化盈利；金融股不用FCFF；等等]
```

⚠️ **分类错误是估值失效的最常见原因。** 对周期股用当期PE估值、对保险公司用FCFF建模，结论必然失真。

**12.0 AI估值诊断（仅 AI 相关公司强制执行，非 AI 公司可跳过）**

**触发条件：** 业务涉及 AI 基础设施、大模型、AI 应用、被 AI 冲击的传统行业、AI 算力等。

**为什么不能跳过：** AI 冲击下，标准 DCF 的参数（利润率目标、Sales/Capital、终值 ROIC）会系统性偏差。
不先做诊断，DCF 输出的数字会严重失真。

**强制执行规则：**

1. 立即调用 `ai-era-valuation` skill，按其 5 个维度（价值链位移、数据护城河、算力成本曲线、
   监管期权、人力资本折旧）+ 双峰分布（如适用）依次执行
2. 输出**参数修正单**——这是 12.1 / 12.6 标准 DCF 建模层的强制输入，**两层顺序不能颠倒**

- 成长价值（简化DCF）
- 结论：当前价格贵/合理/便宜

**12.2 相对估值（必须完成）：**
- PE/PB/PS/EV-EBITDA（当前值 vs 历史均值 vs 行业均值）

**12.3 同业对比（peer-compare skill）：**
- 至少对比3家同业竞争对手的核心估值指标
- ⚠️ 保险股P/EV必须用**寿险EV**，不能用集团EV（两者差一个数量级！）
  - 寿险EV ≠ 集团EV（集团EV含银行等非寿险业务）
  - 计算示例：集团P/EV = 股价 ÷ 集团每股EV（83.07元）
  - 计算示例：寿险P/EV = 股价 ÷ 寿险每股EV（寿险EV ÷ 总股本）
  - 两者意义不同：集团P/EV体现控股折价，寿险P/EV体现核心业务估值

**12.4 SOTP分部估值（sotp-valuation skill，如适用）：**
触发条件：公司为综合性金融集团、多元化产业集团、控股型公司、或新旧业务并存公司。

必须完成的步骤：
1. **业务拆分**：列出各业务板块的核心指标（EV/净资产/AUM等）
2. **板块估值**：各板块分别选择估值方法（P/EV→寿险，P/B→银行，P/AUM→资管，P/S→新业务）
3. **加总计算**：Σ各板块估值
4. **控股折价调整**：综合金融集团通常打8~9折（折价10%~20%）
5. **敏感性分析**：给出不同假设下的估值区间
6. **结论对比**：与当前市值对比，判断低估/高估方向

**SOTP输出格式（必须包含）：**

| 寿险及健康险 | EV=X亿 | P/EV | 0.8x~1.2x | X亿 |
| 银行 | 净资产=X亿 | P/B | 0.4x~0.8x | X亿 |
| 产险及其他 | … | … | … | … |
| **合计** | | | | **X亿** |
| 控股折价 | 8~9折 | | | X亿 |
| **集团合理市值** | | | | **X亿** |

**⚠️ 重要区分：**
- 低估来源 = 控股集团折价 + 银行PB低 → 不是"寿险便宜"
- 寿险P/EV（1.0x~1.2x）往往并不比同业便宜，真正便宜的是控股结构

**保险股特殊规则：**
保险股不用DCF，用P/EV估值。因为保险公司有大量浮存金，DCF的现金流假设不适用。
- 寿险EV：保险公司清算价值，包含有效业务价值（VIFB）+ 经调整净资产（ANA）
- 集团EV：包含寿险EV + 银行净资产 + 产险净资产 + 其他业务 - 集团债务
- P/EV = 市值 ÷ EV，P/EV < 1说明市值低于内含价值（有安全边际）

**12.5 估值术语解释（强制要求，所有类型公司均适用）**

本步骤使用的所有专业术语，**每个术语在报告中首次出现时，必须立刻用括号附上大白话解释**。不得假设读者有财务背景。

**必须解释的核心术语清单（用到哪个解释哪个）：**

| 术语 | 大白话解释模板 |
|-----|--------------|
| DCF / 折现现金流 | "即把公司未来每年能赚到的钱，按一定的折现率折算成今天的价值，加总后即为公司的内在价值" |
| FCFF | "自由现金流（Free Cash Flow to Firm）：公司经营赚来的钱，扣除维持和扩张业务所必须投入的钱，剩下的才叫自由现金流——是真正属于全体资金提供者（股东+债权人）的钱" |
| WACC | "加权平均资本成本（Weighted Average Cost of Capital）：公司向股东和债主融资的综合成本，相当于"公司必须赚到这个回报率，才算没亏待出资人"" |
| 终值 / Terminal Value | "DCF预测期结束后，假设公司以稳定速度永续经营所产生的价值，通常占总估值的60–80%，是估值中最大的不确定性来源" |
| EV / 企业价值 | "Enterprise Value：相当于"买下整个公司要花多少钱"，等于市值加上净负债（债务减现金）" |
| Beta | "衡量这只股票相对大盘的波动幅度：Beta=1表示跟大盘同步；Beta=1.5表示大盘涨1%它涨1.5%，风险更高" |
| ERP / 股权风险溢价 | "投资者选择买股票而非无风险国债，要求额外获得的那部分回报补偿，全球均值约5–6%" |
| Sales-to-Capital Ratio | "每投入1元资本能带来多少元收入，衡量公司的资产轻重：比值越高=越轻资产，钱生钱效率越高" |
| ROIC | "投入资本回报率（Return on Invested Capital）：公司每投入100元运营资本，能赚回多少钱。ROIC持续高于WACC，意味着公司在真正创造价值" |
| P/EV | "市净内含价值比：只用于保险公司。P/EV = 市值 ÷ 内含价值，P/EV < 1意味着你用低于保险公司"清算价值"的价格在买它" |
| EPV | "盈利能力价值（Earnings Power Value）：假设公司不增长、只维持现状经营，应该值多少钱。是估值下限的参考锚" |

**报告写法示例（正确格式）：**
```
用WACC（加权平均资本成本，即公司综合融资成本，
本次取值9.2%，推导见下方12.1）对未来现金流进行折现……
```

**12.6 估值计算过程展示（强制要求）**

估值报告不能只给结论，**必须展示推导过程**，让读者能够验证和质疑每一个数字。

**每个估值模块的计算必须包含以下三层：**

**第一层：关键参数说明**
列出所有输入假设及其来源/依据（不能凭空填数字）：
```
参数设定依据：
- 无风险利率 Rf = 2.3%（来源：2025年4月中国10年期国债YTM）
- 行业Unlevered Beta = 0.82（来源：damodaran-valuation assets/betas.xls，软件行业）
- 目标D/E = 15%（来源：公司近三年平均资本结构）
- ERP（A股默认，强制含 CRP）= 5.15%
    = 美国成熟市场 ERP 4.20% + 中国 CRP 0.95%
    （来源：Damodaran ERPApril26.xlsx + ctrypremApr26.xlsx）
→ 股权成本 Ke = 2.3% + 0.82×(1+0.85×15%)×5.15% ≈ 7.0%
→ 税后债务成本 Kd×(1-t) = (2.3%+1.2%)×(1-25%) = 2.6%
→ WACC = 7.0%×85% + 2.6%×15% ≈ 6.4%
```

> ⚠️ **A 股 ERP 必须包含中国 CRP**：所有 A 股公司默认 ERP = 美国 ERP + 中国 CRP；
> 仅对海外收入占比 > 50% 的公司才按地域加权。忽略 CRP 会让 Ke 系统性低估 70–100bps。
> 央企/大型国企的 ERP 调整规则见 damodaran-valuation/SKILL.md "国企特别处理"章节。

**第二层：年度预测展示**

**第三层：终值与股权价值推导**
```
终值计算：
  终值增长率 g = 2.3%
  ✅ g ≤ min(名义GDP长期增速, 无风险利率 Rf=2.3%)：满足
  ✅ g/Rf = 1.0 ≤ 1：满足
  理由：长期看名义 GDP 应收敛于 Rf；A 股当前 Rf ≈ 2.0–2.3%

  终值 FCFF = 2034 年 FCFF × (1+2.3%) = XX 亿

  终值现值 = XX 亿 ÷ (1+6.4%)^10 = XX 亿
  终值占比 = 终值现值 ÷ EV = X% （强制 < 80%；> 80% 必须重新审视预测期长度或终值假设）

  1. 预测期平均 g_revenue        = X.X%
  2. 预测期平均 ROIC            = X.X%
  3. 隐含 Reinvestment Rate (RR) = g / ROIC = X.X%
  4. 实际 RR = (CapEx - D&A + ΔNWC) / NOPAT 平均 = X.X%
  5. 偏差 = 隐含 RR - 实际 RR = X.X ppt

SBC（股权激励）处理（强制声明）：
  SBC（年化）= X 亿（占净利润 X%）
  处理方式（二选一，禁止"既不扣费用又不稀释"）：
    ☐ 路径 A：在 EBIT 中扣除 SBC 作为真实经济成本，FCFF 用基础股数
    ☐ 路径 B：EBIT 不扣 SBC，但用全摊薄股数（含 RSU/期权 TSM/可转债）
  本估值采用：[路径 A / 路径 B]，理由：____

企业价值汇总（含正确的股权桥接）：
  预测期 FCFF 现值之和  = XX 亿
  + 终值现值          = XX 亿

  - 集团非核心责任   = XX 亿（如未确认的养老金、或有诉讼、表外担保）

  = 归母股权价值      = XX 亿

  = 每股内在价值     = XX 元

**12.7 历史价格验证（强制步骤，所有公司必须执行）**

估值结论不能悬浮在真空里。一个 PB=0.6x 或内在价值 5 元的结论，如果历史上从未出现过对应的成交价格，就必须解释"为什么这次会到那里"——否则结论只是一个内部自洽的数学游戏。

**⚠️ 本步骤是 12.6（估值计算过程展示）的必要补丁，不可跳过、不可合并。**

**12.7.1 基本要求（所有公司）**

完成估值计算后，必须将基准情景的估值结果与历史成交价做可行性核对：

| 核对项 | 要求 |
|-------|------|
| 基准估值 vs 历史 PB 区间 | 基准估值对应的 PB 是否在历史 PB 的区间内？ |
| 基准估值 vs 3/5年均价 | 基准估值与 3 年、5 年长期均价的偏离幅度是多少？ |
| 极端估值 vs 历史极值 | 最乐观/最悲观情景对应的估值，历史上是否出现过？ |

**判断规则：**
- 基准估值在历史区间内 → 注明"基准估值处于历史 XX%-XX% 分位，可接受"
- 基准估值低于历史最低 → **强制解释原因**（如"ROE 将首次跌破 XX%"或"行业结构发生根本变化"）
- 基准估值高于历史最高 → **强制解释原因**（如"估值方法论差异"或"隐性资产未被市场定价"）
- 数据不足（上市<3年） → 注明"历史数据不足以支撑坐标系判断"，不强行下结论

**12.7.2 金融公司专项：ROE-PB-隐含 Ke 历年表**

**适用范围：** 银行、保险、券商、信托等金融服务公司（即 12.-1 分类中的 D 类）。

> ⚠️ **公式适用边界（必读）：** 隐含 Ke 公式 `Ke = g + (ROE - g)/PB` 隐含了 Gordon Growth 的假设
> （ROE 永续稳定、派息率稳定、ROE > g）。在以下场景下公式失效，仅作为参考，不能作为决策依据：
> - **ROE 接近或低于 g 时**：分子 (ROE - g) 趋零或为负，Ke 失去经济意义
> - **ROE 多年波动 > 5pp**（典型如券商，历史 ROE 在 1–20% 间剧烈波动）：单年隐含 Ke 序列从负到 30% 不等，
>   均值/中位数都失去稳定性
> - **该公司处于亏损或恢复期**：BPS 被一次性损益扭曲
>
> 上述任一情形成立时，请改用 **P/B vs ROE 散点图**：横轴 ROE，纵轴 PB，把历史每个年度作为
> 一个点；用线性/对数回归得到"该公司在不同 ROE 水平下市场愿意给的 PB"，再判断当前 PB 是否
> 处于回归带的低位/高位/外面。这种方法对券商、有色金属、有色金融等高 ROE 波动公司更稳健。

金融公司必须建立完整的历史估值坐标系表，逐年回溯：

| 年份 | ROE% | BPS（元）| 年均价（元）| 历史 PB | 隐含 Ke% |
|------|------|---------|-----------|---------|---------|
| 2008 | X.XX | X.XX | X.XX | X.XX | X.X% |
| 2009 | X.XX | X.XX | X.XX | X.XX | X.X% |
| ... | ... | ... | ... | ... | ... |
| 2025 | X.XX | X.XX | X.XX | X.XX | X.X% |

**隐含 Ke 计算公式：**
```
隐含 Ke = g + (ROE - g) / PB
其中 g = 3%（固定），ROE 和 PB 取该年度实际值
```

**这张表必须揭示的三件事：**

**第一，PB 的历史区间和地板。** 最低 PB 出现在哪年、是多少。如果历史上 ROE 跌到 2% 时 PB 跌到 0.7x，就能判断"当前 PB=0.95x 只是接近底部区域而非底部本身"。

**第二，隐含 Ke 的时间序列。** 历史均值隐含 Ke 集中在什么水平？如果历史隐含 Ke 持续低于框架 Ke（如 5% vs 9%），说明市场长期以更低的折现率给券商定价，这需要解释（牌照溢价？政策托底？），并且框架 Ke 的适用性需要讨论。

**第三，ROE 与 PB 的对应关系。** 当 ROE 在 6-7% 时，市场历史上给过什么 PB？这直接检验基准情景估值在历史上有没有出现过对应的定价。

**数据来源：**
- 年均价：新浪财经日 K 线 API（前复权），`https://money.finance.sina.com.cn/quotes_service/api/json_v2.php/CN_MarketData.getKLineData?symbol=sh601555&scale=240&ma=no&datalen=3000`，按年分组取算术平均

**12.8 市场预期诊断（强制步骤，所有公司必须执行）**

绝对估值告诉你**"公允价值的下限"**——但市场长期支付以下溢价：(1) 流动性溢价、(2) 网络效应/平台溢价、
(3) 机构必配带来的指数权重溢价、(4) 未充分计入的可选性（optionality）。这些累计可达 15–40%，
是 Damodaran 估值常常显著低于市价的根本原因。

本节回答三个互补的问题：

1. **市场当前在为什么定价？**（反向 DCF）
2. **市场对这只股票历史上的"舒适区"在哪里？**（历史隐含 g 分位）
3. **当前位置在自身历史和市场整体中如何？**（多重数 z-score + 相对市场分位）

**12.8.1 反向 DCF：市场在为什么定价？**

固定 12.6 的 WACC、终值 g、终值 ROIC、利润率路径，**反向求解**使 DCF = 当前市值的关键变量
（通常是收入 CAGR 或终值利润率）。

```
| 指标                | 我的基础假设 | 市场隐含       | 差异      |
|--------------------|-----------|------------|---------|
| 未来 10 年收入 CAGR | X.X%      | X.X%       | ± X.X ppt|
| 终值 EBIT 利润率    | X.X%      | X.X%       | ± X.X ppt|
| 终值 ROIC           | X.X%      | X.X%       | ± X.X ppt|
| 商业化期 Sales/Capital | X.X    | X.X        | ± X.X    |
```

**叙事翻译（强制写出 ≥ 200 字）：**
> 市场目前在为 ____（具体故事）____ 定价。
> 我的判断是：市场过于乐观 / 悲观 / 合理，因为 ____。
> 隐含差异 X.X ppt 意味着：若我的判断正确，未来 12–24 个月的市场重估方向是 ____。

可使用本 skill 提供的脚本：`scripts/reverse_dcf.py`（见后）

**12.8.2 历史隐含增长率分位（10 年回看）**

`implied_g(t)` 时间序列：

```
| 时点    | 当时股价 | 当时一致预期 EPS/FCF | 反向求解 g_market(t) |
| -10 年  |        |                  |                  |

| -3 年   |        |                  |                  |
| -1 年   |        |                  |                  |
| 当前    |        |                  | ★ 当前值          |

计算分位：
- P25（市场悲观区，胜率参考下沿）= ___ %
- P50（中性区）= ___ %
- P75（市场乐观区，胜率参考上沿）= ___ %

**12.8.5 历史分位法的三个边界警告（必读，避免误用）**

> ⚠️ **警告 1：基本面拐点 → 历史分位失效**
> 诺基亚 2008–2010 历史分位一直显示"严重低估"，但其实是护城河永久性塌陷。
> 出现以下任一信号时，**禁止依赖历史分位**：
> - 核心业务关键指标连续 4 个季度趋势性恶化（不是周期，是趋势）
> - 行业出现颠覆性新进入者（传统燃油车 vs Tesla）
> - 监管/技术范式发生不可逆变化
>
> 此时必须重做基本面分析，并用"新均衡 DCF"重建估值。

> ⚠️ **警告 2：宏观流动性范式切换 → 必须校准 Rf 和分位**
> 茅台 2017–2021 高分位是合理的（LPR/Rf 持续走低、北上资金净流入、机构必配）；2022 后流动性
> 收缩，相同分位的预期收益完全不同。
> **校准规则：当前 Rf 与历史分位区间内 Rf 之差 > 100bps 时，分位需重新计算或重新解读。**

> ⚠️ **警告 3：样本量不足 → 不出 12.8.2/12.8.3 表**
> - 上市 < 7 年：用同行业 4–5 家成熟公司的均值分位作交叉参考，本股票不出独立历史分位表
> - 上市 < 3 年：禁止使用历史分位法；同时按 position-management 规则禁止重仓

#### 13.0 反向论文 / Pre-mortem（强制，写在评分之前）

**触发问题（必答）：** 如果这只股票一年后跌了 50%，最可能的原因是什么？

写出 **3 个 kill points**，每个必须包含以下要素：

```
| Kill Point | 触发信号（量化） | 当前发生概率 | 一旦发生我应该如何反应 |
|----------|--------------|----------|--------------------|
| #1: ____ | 例如"扣非净利润连续 4Q YoY <-15%" | X% | 例如"立即清仓 50% / 加仓 / 不动" |
| #2: ____ | ... | X% | ... |
| #3: ____ | ... | X% | ... |
```

**写法要求：**
- 每个 kill point 必须**具体到可观察、可量化的指标变化**
- 概率不能仅给"低/中/高"——必须给数字（即使粗略也比模糊好）
- 反应是**预先承诺**——一旦发生应避免临时讨论"要不要卖"
- 三个 kill points 不能高度同质（如三个都是"宏观下行"，等于只写了一个）

> "如果你不能在买入之前写出三个让你卖出的具体理由，你大概率没有真正理解这只股票。"

#### 13.1 能力圈自评（强制）

```
□ 我对该行业的研究时长：____ 年/月
□ 我能用 3 句话向小白解释这家公司在做什么吗？☐ 能 ☐ 不能
□ 我跟踪过该公司过去 ___ 年的所有年报与半年报吗？
□ 我能识别出 3 家可比同业的关键差异吗？☐ 能 ☐ 不能
□ 我有过该行业的产品体验/客户访谈/产业链朋友吗？☐ 有 ☐ 无

满足项 / 5 = ___ 项
```

**能力圈自动调整规则：**
- ≥ 4 项满足：评级和仓位上限按下方 13.2 评分表正常应用
- 仅 3 项满足：综合评级最高 **B 级**，单一仓位上限自动 **÷2**
- ≤ 2 项满足：综合评级最高 **C 级（观察仓 ≤ 5%）**，禁止重仓

⚠️ **能力圈外的"高确定性"是错觉**——任何超额收益都不应来自能力圈外。

| 维度 | 权重 | 得分(1–10) | 说明 |
|------|------|-----------|------|
| 商业模式 | 22% | | |
| 护城河   | 28% | | |
| 管理层（含 6.5 步回测）| 18% | | |
| 财务健康 | 13% | | |
| 估值     | 10% | | （见下方否决规则）|
| 流动性/微观结构 | 5% | | （日均成交、纳入指数情况、北上持股变化）|
| 反向论文 kill points 不发生概率 | 4% | | |
| **综合** | 100% | | |

**估值否决项（强制，覆盖综合得分）：**

```
若 内在价值 / 当前股价 < 0.85（即估值高估 > 15%）
   → 综合评级最高 B 级，单一仓位最高 5%（观察仓）
若 内在价值 / 当前股价 < 1.0（即估值高估 0–15%）
   → 综合评级最高 A 级，单一仓位最高 15%
若 内在价值 / 当前股价 ≥ 1.0（即估值合理或低估）
   → 评级和仓位按 position-management 标准应用
```

⚠️ **安全边际是基石**——一家估值显著高估的公司，即使护城河再宽也不应重仓。
但若 12.8 节市场预期诊断显示当前隐含 g 处于历史低位（即"市场比你悲观"），
即使绝对估值不达标，也可在第 15 步触发器规则下进入观察仓或试仓（具体见 Step 15）。

#### 13.3 投资结论与仓位建议

**必须给出：**
- 确定性分级（S/A/B/C）—— 综合得分 × 能力圈调整 × 估值否决 × 反向论文风险后的最终评级

- 具体买入区间
- 加仓触发条件（基本面 + 价格双重）
- 止盈止损条件
- 与 12.8 市场预期诊断结论的呼应（市场是否在为同样的故事定价？）

### 第十五步：操作触发器（Operation Triggers）

**最低字数：500 字**
**触发：** 所有公司都要做（即使绝对估值不达标也可能存在合理操作时机）

绝对估值告诉你**"公允价值的下限"**，但市场长期为流动性、网络效应、机构必配等因素支付溢价。
本步骤把"基本面 + 历史分位 + 估值"三层信号合并为可执行的操作纪律，避免"永远等到完美估值"。

**触发器规则（A–E 五类）：**

```
A. 估值面 + 历史分位双低（12.8.4 四象限的 ★★★ / ★★ 区）
   触发：绝对估值低估 + 历史隐含 g 处于 P25 以下 + 反向 DCF 显示市场比你悲观 ≥ 3 ppt
   操作：可加仓至确定性分级允许的上限，无需额外催化剂

B. 估值面合理 + 强基本面催化（满足任一）
   - 财报超预期且 12 个月内还有 ≥ 2 次大概率超预期
   - 重大产品/订单落地（已在 6 个月内确定）
   - 监管/政策清障（垄断、合规、上市等）
   - 大股东/管理层显著增持（≥ 流通市值 0.5%）
   操作：试仓，仓位上限 ≤ 满仓的 50%
   止损：-15% 强制评估论文是否仍成立

C. 估值面合理偏高 + 双重确认（绝对估值高估 0–15%）
   - 基本面拐点向上（≥ 2 个季度数据支持）
   - 技术面破年线/突破密集筹码区
   - 机构持仓比例从历史低位回升
   操作：跟随仓位（不超过中等确定性的下限），止损纪律严格（-15% 出局）

D. 估值面高估，仍可"对冲式持有"（绝对估值高估 15–30%）
   - 仅当原有仓位的成本远低于当前内在价值时
   - 操作：不卖，但停止加仓；股价若超过历史溢价 P85 → 减仓 1/3

E. 任何情况下的禁止信号（必须减仓至观察仓或清仓）
   - 反向 DCF 显示市场隐含 g > 你的 g 5 ppt 以上
   - 当前估值 + 历史分位均在 P85+
   - 13.0 反向论文中任一 kill point 触发
   - 财务造假或重大违规暴露
```

**强制输出格式：**

```

  - 建仓节奏：____ 
  - 加仓触发：____ 
  - 减仓触发：____ 
  - 止损线：____ 
  - 仓位上限：____ %
```

```
用户输入：深度研究 XXXXXX（股票代码/名称）
                    ↓
[第零步] 准备工作
  ├── 下载最新年报PDF        (annual-report-downloader)
  ├── 下载上一年年报PDF
  ├── 提取PDF财务数字        (pdf-verifier --extract-only)
  ├── 获取实时行情            (AKShare)
  └── 获取五年历史数据        (AKShare)
                    ↓
[第1-11步] 按框架逐步撰写
  每步完成后自检：字数是否达标？关键词是否覆盖？
                    ↓
[第十二步] 估值分析
  ├── 12.-1 公司类型分类（强制前置）→ 确定估值路径
  ├── 12.0 ai-era-valuation（AI相关公司）→ 输出参数修正单
  │     └─ 五个维度：价值链/数据护城河/算力成本/监管期权/人力资本
  ├── 12.1–12.4 按分类结果执行对应估值方法
  │     └─ damodaran-valuation（建模层）→ 接收修正单参数 → DCF建模
  ├── 12.5 术语解释（首次出现即注释，不得假设读者有财务背景）
  ├── 12.6 计算过程展示（参数来源 + 逐年预测表 + 终值推导 + 股权价值汇总）
  └── 12.7 历史价格验证（强制：估值结果 vs 历史成交价可行性核对，金融公司需建ROE-PB-隐含Ke历年表）
                    ↓
[第十三步] 投资建议+仓位管理
                    ↓
[第十四步] 数据来源
                    ↓
[第十五步] PDF数据核验
  发现❌错误 → 修正 → 重新核验
                    ↓
[第十六步] 质检脚本
  不通过 → 返工 → 重新质检
                    ↓
[输出] 完整深度研究报告（预计20-30页）
  present_files 提供给用户
```

**估值层强制串联规则：**
- ai-era-valuation 先跑（诊断层）→ damodaran-valuation 后跑（建模层）
- damodaran-valuation 的参数输入 = ai-era-valuation 的输出修正单
- 两层顺序不能颠倒，颠倒则AI相关公司DCF参数系统性偏差

## 常见偷工减料行为（禁止）

| 偷懒行为 | 后果 |
|---------|------|
| 跳过某步 | 报告作废重写 |
| 字数不足 | 该步骤扩写至达标 |
| 不调用skill | 该步骤重做 |
| 用记忆数据 | 全部财务数字重新拉取核验 |
| 异常波动不解释 | 第五步重做 |
| 不做PDF核验 | 财务数据不可信，第三四步重做 |
| 五年数据不完整 | 第三步重做 |
| 估值不做同业对比 | 第十二步重做 |
| 第十三步无具体数字 | 第十三步重做 |
| **跳过12.-1公司类型分类** | **第十二步全部重做，分类错误导致方法论失效** |
| **估值术语不解释** | **第十二步补写12.5术语解释，直到每个术语都有大白话注释** |
| **只给估值结论不展示计算过程** | **第十二步补写12.6，逐年DCF表格和终值推导必须完整呈现** |
| **跳过12.7历史价格验证** | **第十二步补写12.7，基准估值从未出现过的价格必须解释原因** |
